HTML

概念

  • WEB 指的是计算机领域的网站及相关技术。
  • WAP 指无线设备的网页协议，也就是手机网站，目前最新的是 H5 手机网页。
  • HTML 指的是超文本标记语言。

基本结构

  • <html> 表示代码是 html，主要包含 head 和 body 两部分。
  • <head> 头部，这部分包含显示在网页标题栏中的标题和其他在网页中不显示的信息。
  • <head> 标签中的信息一般是不显示出来的,但是记录了你这个 HTML 文件的很多有用的信息。
  • <title> 设置 HTML 文档的标题。
  • <meta> 元标签。描述网页文档属性。
  • <meta charset="UTF-8"> 网页文档使用 UTF-8 方式编码。
  • <body> 主体部分，显示页面中的文字图像连接等等。
  • <body> 标签之间的信息，是要在网页中展现出来的。
  • <!DOCTYPE html> 表示该页面采用了 HTML5 标准，这样做可以增强页面的兼容性，降低对浏览器的依赖性。
  • 不加 <!DOCTYPE html>，表示页面采用浏览器本身的解析标准，这样会造成页面在不同浏览器出现不同的显示效果。

表单标签

  • <form>定义可供用户输入的HTML表单。如果有<form>标签，在点击提交铵钮时，浏览器自动收集参数，并打包一个http请求到服务器，完成表单提交。在这一过程中，浏览器会根据method的不同，将参数编码后，放在urI中(get)，或者放在请求的data中(post)。然后发送到服务器。如果没有form，post方式的提交需要使用ajax手工完成。get方式的提交需要自己拼接url。

    • action规定当提交表单时向何处发送表单数据。
    • name规定表单的名称。
    • method规定用于发送 form-data 的 HTTP 方法。参数 get 或 post。
    • enctype规定在发送表单数据之前如何对其进行编码。参数如下：
      • application/x-www-form-urlencoded
        • 当请求方式为 get 时候，浏览器用x-www-form-urlencoded的编码方式把form数据转换成一个字串（name1=value1&name2=value2…），然后把这个字符串拼接到url后面，用?分割，加载这个新的url。
      • multipart/form-data
        • 浏览器会把整个表单以控件为单位分割。并为每个部分加上Content-Disposition(form-data或者file),Content-Type(默认为text/plain),name(控件name)等信息，并加上分割符(boundary)。规定数据是以二进制编码的方式传输。

CSS基础

CSS书写方式

  • 内联样式，在HTML元素中使用style属性。
  • 页内样式，在HTML文档头部 <head> 区域使用 <style>标签 来包含CSS样式。
  • 外部样式，在外部 CSS 文件中书写CSS样式，然后通过<link>标签引用。

CSS书写规则

  • CSS样式的书写是通过键值对的方式书写。通过分号 “ ; ” 来分割不同的CSS属性。
  • CSS样式可以理解为编程中的自定义函数。所以末尾不用加分号。
  • 内联样式的优先级别最高。样式的优先级别默认是跟浏览器的执行顺序有关。CSS代码越靠前，其优先级越低。

CSS单位

em

em相对长度单位。相对于当前对象内文本的字体尺寸。

rem

rem是指相对于根元素的字体大小单位，一旦根元素（html标签）的font-size 变化，那么整个网页中运用到rem单位的元素也会随之改变。浏览器默认字体大小是16px，那么1rem等于16px。

元素分类

  • 块级元素
    • 自己单独占一行。能随时设置宽度和高度。
    • 块级标签之间换行符不会保持一定间距。
    • 块级元素不设置长宽，则它的长宽默认由子元素撑开。
    • 常见块级元素
      • ul
      • li
      • form
      • div
      • p
      • h1-h6
      • hr
  • 行内元素
    • 不会自己独占一行。不可以设置宽高。
    • 内联标签之间换行符会保持一定间距。
    • 常见内联元素
      • a
      • br
      • img
      • label
      • span
  • 行内块级元素
    • 可以独占一行，可以设置宽高。
    • 常见行内块级元素
      • input
      • select
      • textarea
      • button

颜色

  • color属性规定文本的颜色。

长宽

  • width属性设置元素的宽度。

  • height属性设置元素的高度。

    • 为什么设置 height 属性 100% 无效？
      • 页面并没有缺省的高度值，所以当你让一个元素的高度设定为百分比高度时，无法获取父元素的高度，也就无法计算自己的高度。
    • 如何设置 body 下子 div 高度为当前浏览器窗口的 50%？
      • html 和 body 标签高度设置 100%，子 div 高度设置 50%。
    • 子元素高度大于父元素高度，父元素设置 overflow:hidden 属性，可以造成一种选项卡与面板连为一体的视觉。

  • min-width属性设置元素的最小宽度。

    • width和min-width的区别？
      • width给块级元素/行内块元素设置固定的宽度，或者固定百分比的宽度。
      • min-width当盒子内部元素宽度小于 min-width的值时，盒子宽度为 min-width的值，当盒子内容宽度大于 min-width的值时，盒子随着内容的增加而被撑大，但是盒子宽度的最小值是设置的min-width的值。

  • max-width定义元素的最大宽度。

浮动和定位

  • float属性定义元素在哪个方向浮动。参数如下：

    • left 元素向左浮动。
    • right 元素向右浮动。
    • 浮动元素会生成一个块级框，而不论它本身是何种元素。所以内联元素设置了浮动元素，就可以设置宽高。
    • 内联元素设置float属性，该内联元素会自动转换为内联块级元素。而块级元素的长宽由内容自动填充，而且不会独占一行。并且会脱离文档流的标准和向上浮动一个层级。

  • position属性规定元素的定位类型。

    • static 默认值。没有定位，元素出现在正常的流中。

    • relative 生成相对定位的元素，相对于其正常位置进行定位。

      • 相对定位注意点：
        1. 不设置平移量，元素不会发生任何变化。
        2. 相对于其正常位置进行定位。
        3. 不会脱离文档流的标准。就算移动元素，也不会影响其他元素。
        4. 不会改变元素的性质，块级还是块级，内联还是内联。

    • absolute 生成绝对定位的元素，相对于 static 定位以外的第一个父元素进行定位。

      • 绝对定位注意点：
        1. 绝对定位的元素相对于距离他最近的开启了定位的父元素进行定位，如果所有元素都没有开启定位，则相对于浏览器进行定位。
        2. 绝对定位的元素会脱离文档流。
        3. 绝对定位元素会改变元素的性质，内联变内联块级，块级的长宽会被内容撑开。并且不独占一行。
        4. 会提升元素的一个高度。
      • float 和 绝对定位 区别？
        • 一个元素浮动时，其他内容（例如：文字）会“环绕”该元素。
        • 绝对定位是使元素完全的脱离文档流，并没有影响到其它元素的内部布局。

    • fixed 生成绝对定位的元素，相对于浏览器窗口进行定位。

      • 固定定位是一种特殊的绝对定位。和绝对定位不同的是，该元素不会随滚动条的滚动而发生位置的改变。
      • 元素的位置通过 “left”, “top”, “right” 以及 “bottom” 属性进行规定。

    • sticky 粘性定位。

      • 在滚动过程中，某个元素距离其父元素的距离达到 sticky 粘性定位的要求时(比如 top: 100px)。position: sticky 这时的效果相当于fixed 定位，固定到适当位置(固定到浏览器距离顶部 100px)。

    • 层级关系：定位元素 >浮动元素 > 文档流中的元素。

背景

  • background简写属性在一个声明中设置所有的背景属性。可以设置如下属性：

    • background-color规定要使用的背景颜色。

    • background-image属性会在元素的背景中设置一个图像。

      • 元素的背景占据了元素的全部尺寸，包括内边距和边框，但不包括外边距。默认地，背景图像位于元素的左上角，并在水平和垂直方向上重复。
      • url('URL')指向图像的路径。
        • 如果这个属性写在css文件中，则相对路径相对与这个css文件，而不是相对与html文件。

    • background-position规定背景图像的位置。

      • xpos ypos第一个值是水平位置，第二个值是垂直位置。
      • x% y%第一个值是水平位置，第二个值是垂直位置。
      • 还可以通过top、 bottom、 left 、 right、 center参数进行设置。

    • background-repeat规定如何重复背景图像。

      • repeat默认。背景图像将在垂直方向和水平方向重复。
      • repeat-x背景图像将在水平方向重复。
      • repeat-y背景图像将在垂直方向重复。
      • no-repeat背景图像将仅显示一次。

    • background-attachment规定背景图像是否（随着滚动条滚动）固定或者随着页面的其余部分滚动。

      • scroll默认值。背景图像会随着页面其余部分的滚动而移动。
      • fixed当页面的其余部分滚动时，背景图像不会移动。
        • 给元素固定图片（最好添加一层蒙版），当页面滚动的时候，会有一种页面穿透图片的感觉。

    • background-size规定背景图片的尺寸。

      • cover把背景图像扩展至足够大，以使背景图像完全覆盖背景区域。背景图像的某些部分也许无法显示在背景定位区域中。
      • contain把图像图像扩展至最大尺寸，以使其宽度和高度完全适应内容区域。
      • length第一个值设置宽度，第二个值设置高度。如果只设置一个值，则第二个值会被设置为 “auto”。

flex（伸缩）布局

  • display:flex;将对象作为弹性伸缩盒显示。

  • display:inline-flex;将对象作为内联块级弹性伸缩盒显示。

  • flex-flow属性是flex-direction和flex-wrap的简写，默认row nowrap。

    • flex-direction用于指定flex主轴的方向，继而决定flex子项在flex容器中的位置。
      • row默认值。灵活的项目将水平显示，正如一个行一样，起点在屏幕左边。
      • row-reverse与 row 相同，但是以相反的顺序，起点在屏幕右边。
      • column灵活的项目将垂直显示，正如一个列一样，起点在屏幕上边。
      • column-reverse与 column 相同，但是以相反的顺序，起点在屏幕下边。
    • flex-wrap属性规定flex容器是单行或者多行，同时横轴的方向决定了新行堆叠的方向。
      • nowrap默认值。规定灵活的项目不拆行或不拆列。
      • wrap规定灵活的项目在必要的时候拆行或拆列。
      • wrap-reverse规定灵活的项目在必要的时候拆行或拆列，但是以相反的顺序。

  • justify-content定义项目在主轴上的对齐方式。

    • flex-start默认值。项目位于容器的开头。
    • flex-end项目位于容器的结尾。
    • center项目位于容器的中心。
    • space-between两端项目对齐（左边和右边没有间隔），其余项目两侧间隔相等。
    • space-around每个项目两侧的间隔相等（左边和右边间隔为x，其余项目两侧间隔为2x）。

  • align-items属性定义flex子项在flex容器的当前行的侧轴方向上的对齐方式。

    • stretch默认值。元素被拉伸以适应容器。如果设置了长度则不拉伸。
    • center元素位于容器侧轴的中心。
    • flex-start元素位于容器侧轴的开头。
    • flex-end元素位于容器侧轴的结尾。
    • baseline元素位于容器的基线上。

  • align-content定义多根主轴的对齐方式。

    • stretch默认值。元素被拉伸以适应容器。如果设置了长度则不拉伸。
    • flex-start元素位于容器侧轴的开头。
    • flex-end元素位于容器侧轴的结尾。
    • center元素位于容器侧轴的中心。
    • baseline元素位于容器的基线上。
    • space-between与交叉轴的两端对齐，轴线之间的间隔平均分布。
    • space-around每根轴线两侧的间隔相等。

  • flex项目的属性：

    • flex属性是 flex-grow、flex-shrink 和 flex-basis 属性的简写属性。

      • flex-grow属性用于设置或检索弹性盒子的放大比率。

        • number一个数字，规定项目将相对于其他项目进行扩展的量。默认值是 0。

      • flex-shrink属性定义项目的缩小比例。默认值都为1，空间不足将等比例缩小。为0不缩放。

      • flex-basis属性定义在分配多余空间之前，项目占据的主轴空间。

        • number一个长度单位（px）或者一个百分比。
        • 如果元素上同时设置了width和flex-basis，那么width 的值就会被flex-basis覆盖掉。
        • width属性只是一个当flex-basis没有被设置时的回退选项。min-width和max-width则是flex-basis的下限和上限。

    • order属性定义项目的排列顺序。

      • number一个整数。数值越小，排列越靠前，默认为0，可以是负值。

    • align-self属性定义flex子项单独在侧轴方向上的对齐方式。

      • auto默认值。元素继承了它的父容器的 align-items 属性。
      • stretch元素被拉伸以适应容器。
      • flex-start元素位于容器的开头。
      • flex-end元素位于容器的结尾。
      • center元素位于容器的中心。
      • baseline元素位于容器的基线上。
      • align-self 不工作的原因？
        • 检查flex弹性盒是否存在align-content属性，align-content优先级大于align-items。

  • transform属性向元素应用 2D 或 3D 转换。该属性允许我们对元素进行旋转、缩放、移动或倾斜。

    • rotate(angle)定义 2D 旋转，在参数中规定角度。

  • transform-origin 属性是用来改变元素变形的原点，一般用来配合旋转来使用最多。

    • 该属性必须与 transform 属性一同使用。
    • 语法：transform-origin: x-axis y-axis z-axis;
      • x-axis定义视图被置于 X 轴的何处。
        • 可能的参数：left、 center、 right、 length、 %。
      • y-axis定义视图被置于 Y 轴的何处。

  • transition属性是一个简写属性，用于设置四个过渡属性：

    • transition-property规定设置过渡效果的 CSS 属性的名称。

      • none没有属性会获得过渡效果。
      • all所有属性都将获得过渡效果。

    • transition-duration规定完成过渡效果需要多少秒或毫秒。

      • time规定完成过渡效果需要花费的时间（以秒或毫秒计）。默认值是 0。

    • transition-timing-function规定速度效果的速度曲线。

      • linear规定以相同速度开始至结束的过渡效果。
      • ease规定慢速开始，然后变快，然后慢速结束的过渡效果。
      • ease-in规定以慢速开始的过渡效果。
      • ease-out规定以慢速结束的过渡效果。
      • ease-in-out规定以慢速开始和结束的过渡效果。
      • cubic-bezier(n,n,n,n)在 cubic-bezier 函数中定义自己的值。可能的值是 0 至 1 之间的数值。

    • transition-delay定义过渡效果何时开始。

      • time规定在过渡效果开始之前需要等待的时间，以秒或毫秒计。

    • 过渡效果添加在哪个元素上，取决于是哪个元素需要过渡效果。不要加在父元素上。添加到需要过渡效果的元素上的伪类也可生效。

    • 只要添加transiton过渡属相，不管元素是否具有transfrom属性，只要元素发生改变，都会获得过渡效果。

自定义动画

  • 通过 @keyframes 规则，您能够创建动画。

  • 创建动画的原理是，将一套 CSS 样式逐渐变化为另一套样式。

  • 以百分比来规定改变发生的时间，或者通过关键词 “from” 和 “to”，等价于 0% 和 100%。

    • 0% 是动画的开始时间，100% 动画的结束时间。

  • 语法：@keyframes animationname { keyframes-selector { css-styles; } }

    • animationname必需。定义动画的名称。
    • keyframes-selector必需。动画时长的百分比。
      • 0-100%
      • from（与 0% 相同）
      • to（与 100% 相同）
    • css-styles必需。一个或多个合法的 CSS 样式属性。

  • animation属性是一个简写属性，用于设置六个动画属性：

    • animation-name规定需要绑定到选择器的 keyframe 名称。

    • animation-duration规定完成动画所花费的时间，以秒或毫秒计。

    • animation-timing-function规定动画的速度曲线。

    • animation-delay规定在动画开始之前的延迟。

      • time可选。定义动画开始前等待的时间，以秒或毫秒计。默认值是 0。

    • animation-iteration-count规定动画应该播放的次数。

      • n定义动画播放次数的数值。
      • infinite规定动画应该无限次播放。

    • animation-direction规定是否应该轮流反向播放动画。

      • normal默认值。动画应该正常播放。
      • alternate动画应该轮流反向播放。

  • animation-fill-mode规定当动画不播放时要应用到元素的样式。

    • forwards在动画结束后，动画将应用动画结束的属性值。

  • animation-play-state规定动画正在运行还是暂停。

    • paused规定动画已暂停。
    • running规定动画正在播放。

格式化上下文

有的盒子呈水平排列，有的盒子却占满整个一行。这是因为不同的盒子，使用的是不同的格式化上下文来布局。

格式化上下文常见的有两种。

  • 块格式化上下文（BFC）
  • 行内格式化上下文（IFC）

BFC

  • BFC 块级(display: block)格式化环境。
    • BFC 是页面上的一个独立容器。
    • BFC属性具有以下特性：
      • 父元素的外边距（margin）不会受子元素影响（即子元素外边距与父元素外边距重叠）。
      • 同级元素的垂直外边距不会合并（原理是子元素外边距不会影响父元素外边距，造成外边距不会合并的效果）。
      • 不会被浮动元素所覆盖（三栏布局）。
      • 可以包含浮动元素。不会造成高度塌陷。
        • 高度塌陷：父元素没有设置高度，高度默认被子元素撑开，一旦子元素脱离文档流，则父元素的高度自动变为 0，造成页面排版混乱。
    • BFC 是元素的隐藏属性，默认是关闭状态，开启BFC的方式：
      • 浮动元素，float 除 none 以外的值。
      • 定位元素，position（absolute，fixed）。
      • display为以下其中之一的值 inline-block, table-cell, table-caption。
      • overflow设置除了visible以外的值（hidden, auto, scroll）。

IE 浏览器常见 Bug

  1. 透明度问题。
    • opacity 属性无效，必须使用 filter: alpha(opacity=80); 设置透明度（取值为0~100）。
  2. IE6 双边局问题。
    • 元素设置左浮动和左外边距（margin-left），元素距离浏览器左边的距离为左外边距 * 2。
    • 设置 _display: inline; 即可解决。
  3. IE6 最小高度为 19 px。
    • 设置 overflow: hidden; 即可解决。
  4. 图片链接产生边框问题。
    • a 标签中嵌套 img 标签会产生边框。
    • 设置 border: none; 即可解决。

渐进增强与优雅降级

渐进增强

优雅降级

一开始针对一个高版本的浏览器构建页面，先完善所有的功能。然后针对各个不同的浏览器进行测试，修复，保证低级浏览器也有基本功能。
